# 员工宿舍管理系统（基础版）

一个面向公司内部的轻量级宿舍管理系统，基于 **Flask + SQLite + HTML/CSS/JavaScript**，支持员工、房间、入住/退宿、报修、违规检查和首页统计看板。

## 1. 项目简介

本项目用于员工宿舍日常管理，特点：

- 中文界面，左侧菜单 + 右侧内容区；
- 支持基础增删改查与查询；
- 包含核心业务规则（床位唯一、房间人数同步、员工住宿状态联动）；
- SQLite 本地数据库，启动即自动初始化演示数据；
- 代码结构简洁，便于后续扩展。

## 3. 安装步骤

> 建议 Python 3.10+。

1. 进入项目目录
   ```bash
   cd Dormitory-Management-System
   ```
2. 创建虚拟环境（可选，但推荐）
   ```bash
   python -m venv .venv
   source .venv/bin/activate
   ```
3. 安装依赖
   ```bash
   pip install -r requirements.txt
   ```

## 4. 启动命令

```bash
python app.py
```

启动后访问：

- http://127.0.0.1:5000/

## 5. 数据库初始化方式

系统提供两种初始化方式：

1. **自动初始化（默认）**
   - 执行 `python app.py` 时会自动创建 `dormitory.db` 并写入演示数据。
2. **手动重置初始化**
   - 在浏览器访问：`/init-db`（例如 http://127.0.0.1:5000/init-db ）
   - 会重建表结构并重新写入演示数据。

## 6. 使用说明

### 6.1 首页 / 数据看板
显示关键统计：
- 员工总人数
- 在住宿舍人数
- 宿舍房间总数
- 总床位数
- 已住床位数
- 空余床位数
- 当前报修未完成数量
- 本月新增入住人数
- 本月退宿人数

### 6.2 员工管理
- 新增 / 编辑 / 删除员工
- 支持姓名、部门、岗位、电话查询
- 住宿状态字段会在办理入住/退宿时自动更新

### 6.3 房间管理
- 新增 / 编辑 / 删除房间
- 支持楼栋、房间号、状态查询
- 当前入住人数由入住/退宿联动维护

### 6.4 入住管理
- 选择未住宿员工办理入住
- 自动校验：
  - 房间不可为“维修中/停用”
  - 房间人数不得超床位总数
  - 同一房间同一床位不可重复入住
- 入住成功后自动：
  - 房间入住人数 +1
  - 员工状态改为“已住宿”
  - 房间状态自动调整（可入住/满员）

### 6.5 退宿管理
- 针对“在住记录”办理退宿
- 退宿成功后自动：
  - 释放床位（入住记录标记为已退宿）
  - 房间入住人数 -1
  - 员工状态改为“未住宿”
  - 房间状态自动调整（可入住/满员）

### 6.6 报修管理
- 新增报修记录
- 可编辑处理状态（待处理/处理中/已完成）
- 若状态为已完成，可记录完成日期

### 6.7 违规检查
- 新增检查记录（卫生/安全/纪律）
- 支持基础查询

## 7. 合理优化说明（相对原需求）

为保证“简单可运行”与“业务正确性”，做了以下优化：

1. 增加了“住宿查询”页面（按员工名查当前住宿），对应“查询某员工当前住宿信息”需求。
2. 入住记录采用 `active` 标记在住状态，避免历史数据丢失。
3. 房间状态中“维修中/停用”不自动被入住人数改写，确保管理状态不被误覆盖。
4. 提供 `/init-db` 快速重置，方便演示与测试。
